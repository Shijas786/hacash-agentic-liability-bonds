# HACD Incubator Issuance Review Checklist

## Project fit

- [ ] Project has a clear reason to use HACD
- [ ] Project is not only a speculative token idea
- [ ] Category is clear
- [ ] Target users are clear
- [ ] Issuer identity / contact is documented

## Stack design

- [ ] Asset type is clear
- [ ] Total supply is clear
- [ ] HACD lots are clear
- [ ] Units per HACD lot are clear
- [ ] Supply math is correct
- [ ] Stack cost in HAC is clear
- [ ] Network fee is disclosed
- [ ] Removal / burn logic is clear
- [ ] Designated address rules are clear, if used

## Launchpad readiness

- [ ] Headline is clear
- [ ] User flow is understandable
- [ ] FAQ covers HACD and HAC requirements
- [ ] Risk disclosure is present
- [ ] No unsupported promises
- [ ] No misleading price floor language
- [ ] Links are verified

## Communication readiness

- [ ] Official X post prepared
- [ ] Educational post prepared
- [ ] Founder-facing post prepared
- [ ] Visual direction prepared
- [ ] Launch date and status are confirmed

## Safety

- [ ] No seed phrases or private keys requested
- [ ] No custody by AI agent
- [ ] No financial advice language
- [ ] No legal guarantee language
- [ ] Human review completed
