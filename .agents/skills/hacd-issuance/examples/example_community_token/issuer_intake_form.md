## 2. Why HACD

- Why should this asset be issued on HACD instead of a normal token contract? StackFire needs credible, visible on-chain formation history. A plain token contract has no PoW cost or scarcity link. HACD gives each lot a real formation cost and a durable state record.
- What does PoW-backed formation add to this project? Formation cost in HAC makes early participation meaningful rather than costless. It sets a visible cost basis reference that regular ERC-20 or similar contracts cannot provide.
- Why does the asset need HACD containers? Each community membership slot is tied to one HACD lot. This makes supply hard-capped and traceable without a centralized database.
- What should users remember after seeing this launch? StackFire is formed, not deployed. Each SFR supply unit exists because a real HACD container was used and a real HAC cost was paid.
