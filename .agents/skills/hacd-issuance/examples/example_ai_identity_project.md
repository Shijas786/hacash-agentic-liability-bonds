# Example: AI Identity Stack Project

## One-liner
A HACD Stack Asset that lets an AI agent use one HACD as a PoW-backed identity and state container.

## Why HACD

AI agents need identities that are not easily reset, copied, or inflated. HACD can act as a scarce PoW-native container where identity, balances, reputation, and asset states can be stacked over time.

## Example structure

- Asset type: hybrid
- 1 HACD = 1 AI identity container
- Stack state may include:
  - agent name
  - supported chain addresses
  - payment routes
  - reputation checkpoints
  - asset ownership states

## Launchpad angle

AI identities should not only be accounts.
They can be formed through PoW-backed containers.
